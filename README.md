# 🌐 FraudInsight AI

### Explainable Financial Fraud Detection Web Application

> *"Intelligent Fraud Detection with Clear Explanations"*

---

## 📌 Project Overview

**FraudInsight AI** is a web-based **Explainable AI (XAI)** system that detects fraudulent financial transactions and explains *why* a transaction was flagged — in plain English.

Unlike black-box models, FraudInsight provides:
- Accurate fraud detection (XGBoost)
- Feature-level explanations (SHAP)
- Human-readable audit reports (LLaMA via Ollama)
- A clean, interactive Streamlit dashboard

---

## 🏗️ System Architecture

```
User (Browser)
     ↓
Streamlit Web App (app.py)
     ↓
Feature Engineering (utils.py)
     ↓
XGBoost Model → Fraud Score (model.pkl)
     ↓
SHAP → Feature Importance
     ↓
LLaMA via Ollama → Plain-English Explanation (ollama_client.py)
     ↓
Results Displayed to User
```

## 📊 Dataset

This project uses the **Kaggle Credit Card Fraud Detection Dataset**:
- URL: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud
- File to download: `creditcard.csv`
- Size: ~144 MB, 284,807 transactions, 492 fraud cases (~0.17%)
- Columns: `Time`, `V1`–`V28` (PCA-anonymised), `Amount`, `Class`

### Downloading the dataset

**Option A — Kaggle web UI**
1. Log in to kaggle.com
2. Go to the dataset page linked above
3. Click **Download** → extract `creditcard.csv`
4. Rename or copy it to `data.csv` inside the project folder

**Option B — Kaggle CLI**
```bash
pip install kaggle
# Place your kaggle.json API token in ~/.kaggle/kaggle.json
kaggle datasets download mlg-ulb/creditcardfraud
unzip creditcardfraud.zip
mv creditcard.csv data.csv
```

## 🚀 Setup & Run

### Prerequisites
- Python 3.10 or higher
- [Ollama](https://ollama.com) installed on your machine

### Step 1 — Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### Step 2 — Install dependencies

```bash
pip install -r requirements.txt
```

### Step 3 — Download the dataset

Follow the dataset download steps above. The file must be at `data.csv` in the project root.

### Step 4 — Train the model

```bash
python train.py --data data.csv
```

This produces `model.pkl` and `feature_names.pkl`. Expected output:
```
[1/4] Loading data from: data.csv
      Rows: 284,807  |  Fraud: 492  |  Legit: 284,315
[2/4] Splitting — test size: 20%, seed: 42
[3/4] Training XGBoost  (scale_pos_weight=578.1)
...
  ROC-AUC : 0.9812
  Model saved → model.pkl
```

Training takes ~1–2 minutes on a modern laptop.

### Step 5 — Start Ollama + LLaMA

```bash
ollama run llama3
```

Wait until you see `>>> Send a message` — the server is now running on `http://localhost:11434`. You can close that terminal; the server keeps running in the background.

### Step 6 — Run the web app

```bash
streamlit run app.py
```

Open your browser at: **http://localhost:8501**

## 🧠 Model Configuration

```python
model = xgb.XGBClassifier(
    n_estimators=300,
    max_depth=6,
    learning_rate=0.05,
    subsample=0.8,
    colsample_bytree=0.8,
    scale_pos_weight=578,    # auto-calculated from class ratio
    eval_metric="auc",
    use_label_encoder=False,
    random_state=42,
)
```

`scale_pos_weight` is set automatically during training as `n_legit / n_fraud` to handle the severe class imbalance.

## 🔌 Ollama API Integration

The app calls Ollama's local REST API via `ollama_client.py`:

- **Endpoint:** `POST http://localhost:11434/api/generate`
- **Model:** `llama3` (configurable in sidebar)
- **Timeout:** 60 seconds (increase in `ollama_client.py` for slow hardware)
- **Stream:** disabled (waits for full response)

If Ollama is not running, the app shows a warning banner and skips the LLaMA explanation — SHAP output is still displayed.

### LLaMA prompt template

```
You are a financial fraud analyst writing a report for a bank auditor.
Analyze the following transaction and explain clearly why it has been flagged.
Be concise (3–5 sentences). Use plain English. Do not use bullet points.

Transaction details:
  - Amount: ₹{amount}
  - Customer's average transaction: ₹{avg_amount}
  - Location: {location}
  - Time of day: {time_of_day}
  - Fraud probability score: {score} ({risk_label} risk)

Key risk factors identified:
{factors}

Write the explanation now:
```
